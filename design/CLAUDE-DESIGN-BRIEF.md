# Claude Design 브리프 — YouTube → Spotify 변환기 (크롬 확장 UI)

크롬 확장(Manifest V3)의 UI를 **프로덕션에 바로 붙일 수 있는 HTML/CSS**로 디자인해줘.
아래 화면·컴포넌트·데이터·제약을 정확히 지키고, **미학 목표(§2)** 를 최우선으로.

---

## 1. 무엇을 만드나
유튜브 "노래 모음" 영상(한 영상 안에 수십 곡)을 분석해 각 곡을 Spotify에서 찾아
**영상 제목과 같은 이름의 재생목록**을 만들어 주는 확장. 매칭이 애매한 곡은 사용자가
후보 중에서 고르거나 직접 검색해 추가한다. 다크 테마 전용.

디자인할 표면은 둘:
- **팝업** (툴바 아이콘 클릭, 폭 **380px 고정**) — 시작점 + 간단 검토
- **전용 탭 페이지** (폭 최대 **600px**, 중앙 정렬) — 진행·결과·검토 풀뷰

---

## 2. 미학 목표 ★가장 중요★
**"노련한 프로덕트 디자이너가 만든 프리미엄 음악 툴."** Spotify 앱의 절제를 갖되,
기본 템플릿보다 **타이포·여백 리듬·디테일에서 확실한 완성도**가 느껴져야 한다.

### 반드시 피해야 할 두 실패 사례
- **❌ "AI 양산형"** — 글라스모피즘, backdrop-blur, 네온 글로우, 장식용 그라데이션·그라데이션 텍스트,
  별가루/파티클, Inter/DM Sans 폰트. (초기 시안이 이 실수를 저질렀음)
- **❌ "초심자형"** — 균일한 회색 박스 + 어디나 똑같은 8px 간격 + 크고 굵은 흰 글씨만.
  깊이도 개성도 없는 상태. (**첨부 `layout-reference.html`(v3)가 지금 이 상태**)

### 목표 지점 (이 사이)
- **타이포그래피가 주인공** — 또렷한 위계, 개성 있는 서체(디스플레이/본문 구분), 정교한 크기·굵기·자간.
- **앨범아트가 유일한 색의 원천** — 콘텐츠에서 나온 색을 절제 있게 활용(예: 재생 중 곡의 아트에서
  추출한 은은한 색조 — Spotify now-playing처럼, 단 과하지 않게).
- **정밀한 간격 리듬과 정렬** — 스캔이 쉬운 밀도, 의미 있는 여백.
- **모든 상태를 디자인** — hover/active/focus/empty/loading/error 각각.
- **절제된 목적 있는 모션** (120–200ms).
- **정체성을 주는 시그니처 디테일 1–2개** (예: "YouTube 원본 → Spotify 후보" 를 표현하는 방식,
  미리듣기/now-playing 트리트먼트).

### 첨부 레퍼런스를 최우선으로
사용자가 함께 첨부한 **레퍼런스 스크린샷의 타이포·색·디테일 감각**을 이 프로젝트에 이식해줘.
(레이아웃이 아니라 "미감"을 가져오는 것)

## 3. 화면 & 상태

### 팝업 (380px)
1. **시작** — 브랜드 헤더 / 감지된 영상 카드(썸네일+제목+채널+곡수) / Spotify 연결 상태·버튼 /
   트랙리스트 미리보기(스크롤) / "재생목록으로 변환" 주요 버튼.
2. **미연결/비지원** — 연결 안내, 또는 "이 영상엔 트랙리스트가 없어요" 빈 상태.
3. **진행 배너** — 변환 중이거나 확인 대기 곡이 있으면 상단에 배너 + "상태 보기"(탭 열기).
4. **간단 검토** — 팝업 안에서도 확인 필요 곡을 후보와 함께 볼 수 있어야 함(전용 탭의 축약판).

### 전용 탭 (≤600px)
5. **진행 중** — 영상 제목 / 진행률 바(N/총) / "탭 닫아도 계속됩니다" 안내.
6. **결과 요약** — 3분할 통계(자동 추가 / 확인 필요 / 못 찾음) + "추가된 N곡"(앨범아트 스택) +
   "Spotify에서 재생목록 열기" 버튼.
7. **확인 필요 리스트** — 각 항목: `YouTube 원본`(곡명·타임스탬프) → `Spotify 후보`(2~5개).
   후보 = 앨범아트 + 곡명 + 아티스트·앨범·연도 + 재생시간 + "추가" 버튼.
   후보 행을 누르면 **Spotify 임베드 플레이어(iframe)** 가 펼쳐져 미리듣기.
   하단에 **직접 검색 입력** + "건너뛰기".
8. **못 찾음 리스트** — 원본 곡명 + 직접 검색 입력(+결과 후보) + 건너뛰기.
9. **완료** — 모두 처리 시.

## 4. 컴포넌트 인벤토리
브랜드 로고/워드마크, 주요/보조/고스트 pill 버튼, 영상 카드, 연결 상태 행,
트랙리스트 행, 통계 타일, 재생목록 열기 버튼(Spotify 로고 포함), 검토 항목 카드,
후보 행(앨범아트 썸네일), 미리듣기 임베드 영역, 검색 입력+버튼, 상태 뱃지(확인필요/못찾음),
진행률 바, 빈/로딩/에러 상태.

## 5. 실제 데이터 형태 (이 필드명으로 바인딩되게)
```jsonc
// 인증
authState = { connected: bool, profile: { id, name, country } }

// 영상 (콘텐츠 스크립트)
video = { title, channel, url,
  tracks: [{ index, time:"3:56", durationSec, artistGuess, titleGuess, label }] }

// 변환 상태 (폴링)
convertState = {
  status: "running"|"done"|"error", videoTitle, total, processed, error,
  playlistUrl,
  added:   [{ id, label, time, score, track }],
  review:  [{ id, label, time, score, candidates: [track] }],
  notFound:[{ id, label, candidates: [track] }]
}

// track (후보/추가곡 공통)
track = { uri, id, name, artists:[string], durationMs, isrc, popularity,
  image /* 앨범아트 URL */, album:{ name, type, releaseDate:"2021-11-04" } }
```

## 6. 메시지 API (JS를 함께 만든다면 이걸로 배선)
`chrome.runtime.sendMessage({type})` — `CONNECT_SPOTIFY` · `DISCONNECT_SPOTIFY` ·
`GET_AUTH_STATE` · `CONVERT{video}` · `GET_CONVERT_STATE`(700ms 폴링) ·
`RESOLVE_REVIEW{itemId, uri}` · `MANUAL_SEARCH{query}` · `CLEAR_CONVERT`.
미리듣기 임베드: `https://open.spotify.com/embed/track/{id}`.

---

## 7. 기술 제약 (반드시 준수)
- **Manifest V3**: 확장 페이지 CSP상 **인라인 `<script>` 금지** → JS는 별도 `.js` 파일로.
  인라인 `<style>`·별도 `.css`는 가능.
- **폰트는 셀프호스팅** 전제(원격 폰트 로드 불가). Pretendard(한/영) + Noto Sans JP 폴백을 기본으로 하되,
  더 개성 있는 디스플레이 서체 제안 가능(웹폰트로 내장 가능한 것).
- 팝업 **380px 고정**, 탭 **≤600px**. 다크 전용. 터치/클릭 타깃 ≥36px.
- 한/일/영 혼용 텍스트가 깨지지 않게(곡명이 日·英, UI는 韓).

---

## 8. 산출물
화면별 **HTML + CSS**(프로덕션 적용용, 위 컴포넌트/데이터 구조 반영).
가능하면 **디자인 토큰**(CSS 변수: 색·타입 스케일·간격·radius)을 먼저 정의하고 그 위에 화면을 구성.
레이아웃은 첨부 `layout-reference.html` 구조를 기준으로 하되 **비주얼은 완전히 새로**.
