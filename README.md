# YouTube → Spotify 변환기

유튜브 노래 모음(플레이리스트) 영상 하나에서 안에 담긴 곡들을 읽어, Spotify에서 찾아
영상 제목과 같은 이름의 재생목록으로 만들어 주는 크롬 확장입니다.

플레이리스트 영상을 Spotify로 옮길 때 곡을 하나하나 검색해 담는 게 번거로워서 만들었습니다.
지금은 v0.8.0이고, 웹스토어에 올리기 전 지인들에게 테스트를 받고 있습니다.

## 동작

설명란 타임스탬프, 챕터, 고정 댓글, 유튜브가 자동으로 인식한 "음악" 섹션 중 하나에서
트랙리스트를 읽습니다. 이 넷 다 없는 영상은 지원하지 않습니다.

각 곡을 Spotify에서 검색해 아티스트와 재생 시간을 비교하고, 확실한 건 자동으로 담고
애매한 건 따로 모아 직접 고르게 합니다. 같은 곡의 발매본이 여럿이면 컴필레이션·리마스터를
빼고 최초 발매를 고릅니다. 표기가 달라도(로마자와 일본어 등) 아티스트가 같으면 같은 곡으로
봅니다.

애매한 곡은 후보를 Spotify 임베드로 미리 들어보고 고르거나 건너뛸 수 있고, 직접 검색해서
넣을 수도 있습니다. 검토를 마치고 재생목록을 만들기 전까지는 Spotify에 아무것도 생성하지
않습니다. 중간에 그만둬도 반쪽짜리 재생목록이 남지 않게 하려고 그렇게 했습니다. 만들어지는
재생목록은 비공개이고, 곡 순서는 원래 트랙리스트 순서를 따릅니다.

## 권한과 데이터

Spotify 로그인은 OAuth Authorization Code + PKCE 방식이고 클라이언트 시크릿을 쓰지
않습니다. 확장은 youtube.com(트랙리스트 읽기)과 Spotify API에만 접근합니다. 토큰은 브라우저
로컬 저장소에만 두고 어디에도 보내지 않습니다.

## 구조

Manifest V3. 팝업, 서비스 워커, 변환 진행·검토용 전용 탭으로 나뉩니다. 매칭과 파싱 로직은
의존성 없는 바닐라 JS(`src/lib/`)로 분리해 두어서 Node로 바로 테스트할 수 있습니다. 테스트는
`chrome`·`fetch`·DOM을 모두 목으로 대체하므로 Spotify를 실제로 호출하지 않습니다.

```bash
for f in test/*.test.js; do node "$f"; done
```

## 설치 (테스트판)

웹스토어 등록 전이라 압축을 풀어 직접 로드해야 합니다.

Spotify 연결은 개발자 대시보드에 등록된 계정만 됩니다(테스트 단계, 최대 25명). 사용할 Spotify
계정 이메일을 제게 먼저 알려주세요.

1. [Releases](../../releases)에서 zip을 받아 압축을 풉니다. (폴더를 지우면 확장도 꺼지니 안 지울 곳에 두세요.)
2. 크롬 주소창에 `chrome://extensions` 를 엽니다.
3. 오른쪽 위 개발자 모드를 켭니다.
4. "압축해제된 확장 프로그램을 로드"를 눌러 압축 푼 폴더를 고릅니다.
5. 툴바에 아이콘이 보이면 됩니다. (안 보이면 퍼즐 조각 아이콘에서 고정하세요.)

## 사용

유튜브에서 노래 모음 영상을 열고, 확장 아이콘을 눌러 Spotify에 연결한 뒤(최초 1회),
"재생목록으로 변환"을 누릅니다. 애매한 곡을 검토·선택하고 "재생목록 만들기"를 누르면
영상 제목과 같은 이름의 비공개 재생목록이 생깁니다.

영상을 클릭으로 이동한 직후 인식이 이상하면 새로고침을 한 번 하세요. 변환 탭을 닫아도
백그라운드에서 계속됩니다. 뭔가 꼬였으면 변환 화면의 초기화로 처음부터 다시 시작할 수 있습니다.
